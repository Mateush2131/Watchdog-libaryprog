- **"Event-Driven Architecture"** — принципы событийно-ориентированного программирования

## Как использовались источники

1. **Начальный этап** — изучение официальной документации для понимания возможностей библиотеки
2. **Проектирование** — анализ примеров из сообщества для выбора архитектуры
3. **Реализация** — поиск решений конкретных проблем на Stack Overflow
4. **Улучшение** — изучение принципов чистого кода для рефакторинга
5. **Документирование** — использование примеров технического письма из документации Python

**Важное примечание:** Все заимствованные идеи и код были адаптированы под конкретные задачи проекта и не являются прямой копипастой. Принципы работы watchdog были изучены и переработаны для создания собственной системы архивации.
